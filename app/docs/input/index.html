Input fields

Input fields are generally labeled with .input. For all bricks.css inputs the labels are included above the field itself and the usage of placeholders is not recommended.

Basic text input

Text inputs come in variety of HTML5-defined fields. The are three ways of displaying the input fields:

I'm extra information provided.
I can even be on multiple lines if needed.
<input type="text" class="input input__text"></input>

<label class="input__label" for="text-with-label">Text field with a label</label>
<input class="input input__text" type="text" name="text-with-label"></input>

<label class="input__label has--tooltip" for="text-with-label-and-tooltip">Text field with a label and extra information</label>
<input class="input input__text" type="text" name="text-with-label-and-tooltip"></input>
<div class="tooltip">I'm extra information provided.<br>I can even be on multiple lines if needed.</div>

Advanced text input

The advanced HTML5 input types are not compatible in all browsers, however they add extra usability for users browsing the site on compatible browsers (e.g. custom keyboards on iOS). They fail peacefully, reverting to the basic input fields. The advanced HTML5 input types are:

Please, enter a number.

Conditional input

Conditional inputs visually hint the user about validation of the data they are putting in. There are three basics patterns that can be mixed with all the fields above. Each field can include a custom validation patters, such as the email field below.

Please, enter a valid email address (e.g. john.doe@gmail.com).
<label class="input__label">Required text field</label>
<input type="text" class="input input__text" required></input>

<label class="input__label">Email field with error</label>
<input type="email" class="input input__text" pattern=".+@.+\..+"></input>
<div class="error input__error">Please, enter a valid email address (e.g. john.doe@gmail.com).</div>

<label class="input__label">Disabled text field</label>
<input type="text" class="input input__text" disabled></input>
